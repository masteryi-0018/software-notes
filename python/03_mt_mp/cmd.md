## 多线程

1. 受到GIL限制，本质上还在一个进程中，同一时刻只有一个线程在执行Python字节码
2. 不需要fork()，线程创建开销小，共享内存和全局的变量，线程之间只有栈和寄存器是不同的
3. 有没有`__name__ == "__main__"`的都可以

## 多进程

1. 不受GIL限制，可以真正并行执行，感觉和手动开多个Python进程一样
2. 进程创建开销大，进程之间通信需要IPC机制（如管道、队列、共享内存等）
3. 对于没有`__name__ == "__main__"`的问题，Windows和Unix/Linux的行为不同：
   - Windows/MacOS：使用spawn，必须使用`if __name__ == "__main__":`来保护代码，否则会导致无限递归创建子进程
   - Linux：使用fork，不会重新import脚本，所以不需要，但推荐使用以保持代码的可移植性和清晰性
   - **对于cuda而言，需要使用spawn来创建子进程**，所以也必须使用`if __name__ == "__main__":`来保护代码
